## Design Context

### Users
個人の開発者が、自分のClaude Code活用スタイルを診断し改善するために使うWebアプリ。
10問のクイズに答え、6タイプの中から最適な活用スタイルを発見する。
コンテキスト: 開発の合間や環境改善の検討時にアクセスする。データはサーバーに送信されない。

### Brand Personality
**3 words**: Professional, Minimal, Approachable

- トーン: 信頼感をベースに、診断という体験に少しの遊び心を添える
- ブランドカラー: コーラル（#da5b38 → #c44a2a グラデーション）をアクセントとして維持
- タイプ別カラー: 青(Explorer)・紫(Architect)・黄(Engineer)・赤(Commander)・緑(Scholar)・ピンク(Visionary) — これらは結果画面で主役になる
- フォント: Geist Sans / Geist Mono（システマチックで読みやすい）

### Aesthetic Direction
- **Visual tone**: ミニマルでクリーンなニュートラルベース。過剰な装飾を避け、情報の明瞭さを優先
- **Motion**: Framer Motionによる控えめなアニメーション（フェードイン、スプリング、ホバーリフト）。機能的な遷移のみ
- **Theme**: Light/Dark両対応。shadcn/uiのニュートラルパレットをベースに、コーラルとタイプ色でアクセント
- **Reference direction**: shadcn/uiの公式サイトのような洗練されたミニマリズム
- **Anti-reference**: 過度にゲーミフィケーションされた診断サイト、ネオンカラー、アニメーション過多

### Design Principles
1. **Content First** — UIは情報を伝えるための器。装飾ではなく、スコア・タイプ・提案が最も目立つこと
2. **Quiet Confidence** — 信頼感は派手さではなく、一貫したスペーシング・タイポグラフィ・コントラストから生まれる
3. **Color with Purpose** — コーラルはCTA・ブランドに、タイプ色は結果画面に。それ以外はニュートラル
4. **Progressive Disclosure** — 情報は段階的に開示する（アコーディオン、タブ、遷移アニメーション）
5. **Accessible by Default** — WCAG AA準拠。コントラスト比4.5:1以上、キーボード操作対応、Reduced Motion対応

### Technical Stack
- Next.js 16 + React 19 + TypeScript
- Tailwind CSS v4 + shadcn/ui (Neutral base)
- Framer Motion for animations
- Chart.js for radar charts
- Storybook 10 for component management
